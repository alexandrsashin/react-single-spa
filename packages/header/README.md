# Header Microfrontend

## Описание

Третий микрофронтенд в виде хэдера, который отображается на маршрутах `/` и `/user`. Содержит:

- Логотип приложения
- Parcel с уведомлениями (иконка-колокольчик)
- Информацию о пользователе и меню действий

## Порт разработки

Работает на порту **3008**

## Компоненты

### Header

Основной компонент хэдера с:

- Логотипом приложения
- Информацией о пользователе
- Выпадающим меню с действиями (настройки, выход)

### NotificationBell (Parcel)

Parcel-компонент с уведомлениями:

- Иконка-колокольчик
- Badge с количеством непрочитанных уведомлений
- Popover с списком уведомлений
- Автоматическое добавление новых уведомлений каждые 30 секунд
- Возможность отмечать как прочитанные и удалять

## Функции

### Уведомления

- ✅ Показ badge с количеством непрочитанных
- ✅ Popover со списком уведомлений
- ✅ Типы уведомлений: info, success, warning, error
- ✅ Временные метки ("только что", "5 мин назад", и т.д.)
- ✅ Отметка как прочитанные
- ✅ Удаление уведомлений
- ✅ Кнопка "Прочитать все"
- ✅ Автоматические тестовые уведомления

### Интеграция с AuthService

- ✅ Отображение информации о пользователе
- ✅ Реакция на изменения состояния авторизации
- ✅ Скрытие при неавторизованном состоянии

### Навигация

- ✅ Кнопка выхода через NavigationService
- ✅ Заглушка для настроек

## Отображение

Header отображается на маршрутах:

- `/` (после авторизации)
- `/user`

Не отображается на:

- `/login`

## Стили

Использует Ant Design для:

- Layout.Header
- Badge, Popover для уведомлений
- Avatar, Dropdown для пользователя
- Typography для текста
- Button для действий

## Тестирование

Для тестирования уведомлений:

1. Авторизуйтесь в системе
2. Новые уведомления будут появляться каждые 30 секунд
3. Кликните на колокольчик для просмотра
4. Используйте кнопки для взаимодействия с уведомлениями
